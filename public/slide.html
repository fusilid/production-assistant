⚠
The Problem
  • Shift managers handle multiple lines under time pressure
  • Critical decisions needed in under 10 minutes
  • Historical incidents & standards spread across disparate systems
  • Missed safety signals lead to personnel risk
  • Tribal knowledge lost at shift handoff
🛡
Safety-First Architecture
  • Deterministic classifier runs before LLM — cannot be bypassed
  • Covers injury, fire, gas leak, arc flash, LOTO, confined space, explosion, entrapment & more
  • Triggered flag overrides any model output after the fact
  • Safety actions lead the Stabilize bucket when triggered
  • EHS lead automatically assigned as action owner
✦
Responsible AI Design
  • Hypotheses, not verdicts — 2–4 ranked candidates with confidence
  • Every claim must cite a source_id from retrieved context
  • LLM output validated against Zod schema before serving
  • API key stays server-side only — never sent to browser
  • Structured JSON output — no free-form hallucination risk
◈
Context Retrieval (RAG)
  • Process Standards — SOPs, thresholds, compliance refs
  • Prior Incidents — same line first, then same plant (max 8)
  • Maintenance Records — work orders, asset history, parts/lots
  • Quality Thresholds — Cpk targets, measurement limits, typical root causes
  • All cited by source_id in LLM response
▤
Analysis Output
  • Stabilize — stop-the-bleeding actions, due "now"
  • Investigate — hypothesis-testing steps, this shift
  • Prevent Recurrence — systemic fixes, this week
  • Each action has: owner role, ETA, linked evidence
  • Draft generator: shift handoff, maintenance request, CAPA, supplier Q&A
⬡
Tech Stack
  • Next.js 16 (Turbopack) — App Router, server actions
  • Claude Sonnet 4-5 via Anthropic SDK — server-side only
  • Zod 4 — end-to-end schema validation
  • Tailwind CSS 4 + shadcn/ui — component library
  • Vitest — unit tests for safety classifier & evals
